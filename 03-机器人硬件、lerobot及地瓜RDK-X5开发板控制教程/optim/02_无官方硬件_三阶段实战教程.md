# Joy-Con 具身智能遥操作实战：从虚拟到现实

本教程专为没有官方 LeRobot 机械臂套件的开发者打造。只需一个 Nintendo Switch Joy-Con 手柄、一台个人电脑、一块 RDK X5 开发板以及一台基础的 3 自由度机械臂，你就能打通**“仿真遥操作 -> 边缘端部署 -> 物理真机控制”**的完整具身智能数据采集链路。

教程分为循序渐进的三个实战篇章。

---

## 💻 实战一：虚拟机械臂 + Joy-Con + 个人电脑
**目标**：在你的主力电脑上跑通底层驱动，并通过 Robosuite 仿真环境体验手柄遥控虚拟机械臂。

### 1.1 环境连通（连接手柄）
1. 电脑打开蓝牙，按住 Joy-Con 侧边的小圆钮进入配对模式。
2. 在电脑蓝牙列表中找到 `Joy-Con (R)` 或 `Joy-Con (L)` 并连接。
3. **（重要！）** 如果你使用的是 Linux (如 Ubuntu) 系统，必须配置底层权限，否则 Python 无法读取手柄数据。打开终端执行：
   ```bash
   sudo apt update && sudo apt install libhidapi-dev
   echo 'SUBSYSTEM=="hidraw", ATTRS{idVendor}=="057e", MODE="0666"' | sudo tee /etc/udev/rules.d/99-joycon.rules
   sudo udevadm control --reload-rules && sudo udevadm trigger
   ```
   **注意：执行完后，必须断开手柄蓝牙并重新连接一次！** Windows 用户可跳过 udev 配置。

### 1.2 安装核心控制库
推荐使用 `conda` 创建隔离的 Python 环境：
```bash
conda create -n robosuite python=3.10
conda activate robosuite

# 1. 安装手柄底层通信驱动
git clone https://github.com/box2ai-robotics/joycon-robotics.git
cd joycon-robotics
pip install -e .
sudo apt install dkms  # Linux 环境下需要
make install
cd ..

# 2. 安装 Robosuite 仿真及手柄控制库
git clone https://github.com/box2ai-robotics/robosuite-joycon.git
cd robosuite-joycon
pip install -r requirements.txt
pip install -e .
```

### 1.3 启动仿真体验遥操作
确保你的电脑有图形化桌面环境。在终端中运行单手柄仿真脚本：
```bash
python robosuite/demos/demo_device_control_joycon_single.py
```
**实战效果**：屏幕上会弹出一个 3D 仿真窗口，里面有一台高端机械臂。此时你推动 Joy-Con 的摇杆，虚拟机械臂的末端就会跟着在 X/Y/Z 轴上移动，按下扳机键可以控制夹爪开合。
*(这就是前沿 AI 论文中用来录制虚拟数据集的真实场景！)*

## 🌐 实战二：虚拟机械臂 + Joy-Con + RDK X5 (边缘计算架构)
**目标**：将 RDK X5 改造为“无线接收基站与 Web 服务器”，在 RDK X5 上无渲染运行算法后台，通过局域网在你的个人电脑浏览器上进行遥控与监控。

### 2.1 部署 RDK X5 边缘服务器
首先，像实战一那样，将 Joy-Con 蓝牙连接到 **RDK X5** 上，并同样配置 `udev` 规则和 Python 环境。

接着，在 RDK X5 上安装前端运行环境 Bun 并准备 Web 服务代码：
```bash
# 在 RDK X5 上执行
curl -fsSL https://bun.sh/install | bash
source ~/.bashrc

cd robosuite-joycon-web
bun install
```

### 2.2 开启双核控制系统 (RDK X5 端)
在 RDK X5 上开启两个终端窗口（都在 `robosuite` 环境下）：

**终端 1（后台物理运算引擎）：**
因为 RDK X5 不适合渲染复杂的 3D 画面，我们运行无渲染的测试服务器：
```bash
python test_server_no_render.py
# 确认不报错后，这代表 RDK X5 已经在 60Hz 监听 WebSocket 了
```

**终端 2（Web 监控面板）：**
```bash
bun run dev --host
```
记录下终端输出的局域网 IP（例如 `http://192.168.x.x:5173`）。

### 2.3 局域网跨端遥操作
1. 回到你的**个人电脑**，打开 Chrome/Edge 浏览器，输入刚才 RDK X5 的局域网 IP。
2. 页面加载出仪表盘后，点击**“连接服务器”**。
3. 按下手里的 Joy-Con 按钮激活权限。
**实战效果**：你的手柄指令被电脑浏览器捕获 -> 变成 WebSocket 数据包飞给同在一个 WiFi 下的 RDK X5 -> RDK X5 在后台完成了 7 维矩阵和物理引擎的计算。这就完成了一个工业级的低延迟遥操作分布架构。

## 🦾 实战三：3 自由度真机 + Joy-Con + RDK X5
**目标**：抛弃虚拟仿真，编写胶水代码，用手柄直接控制现实桌面上由大臂、小臂和夹爪组成的 3 自由度真实机械臂！

### 3.1 硬件连接与理解
- **你的硬件**：底座通常固定不转。1 个舵机控制大臂（上下俯仰），1 个舵机控制小臂（上下俯仰），1 个舵机控制前端夹爪。
- **通信方式**：将舵机控制板通过 USB 串口或 GPIO 连接到 RDK X5。

### 3.2 编写胶水代码 (直接映射法)
由于没有官方 LeRobot 的完整 SDK 支持，我们采用“最硬核也最简单”的直接摇杆映射法。
在 RDK X5 上新建一个 Python 脚本 `custom_arm_control.py`，核心逻辑如下：

```python
import time
# 导入你的手柄底层库 (由 joycon-robotics 提供支持)
from joycon import JoyCon

# ================================
# 这里替换为你真实机械臂的串口发送函数
def set_servo_angle(servo_id, angle):
    print(f"发送指令: 舵机 {servo_id} 转动到 {angle} 度")
    # serial_port.write(...)
# ================================

def main():
    # 连接手柄
    jc = JoyCon()
    print("手柄已连接，开始遥控真实机械臂！")
    
    # 设定舵机初始安全角度
    arm_base_angle = 90  # 大臂
    arm_elbow_angle = 90 # 小臂
    gripper_state = 0    # 夹爪 0=松 1=紧

    while True:
        # 获取最新状态
        state = jc.get_status()
        
        # 1. 左摇杆上下推 -> 控制大臂
        left_y = state['analog-sticks']['left']['vertical']
        if abs(left_y) > 0.1: # 消除死区
            arm_base_angle += left_y * 2  # 乘以系数调节速度
            
        # 2. 右摇杆上下推 -> 控制小臂
        right_y = state['analog-sticks']['right']['vertical']
        if abs(right_y) > 0.1:
            arm_elbow_angle += right_y * 2
            
        # 3. ZR 扳机键 -> 控制夹爪
        if state['buttons']['right']['zr']:
            gripper_state = 1
        else:
            gripper_state = 0
            
        # 角度限位保护 (防止机械臂撞坏自己)
        arm_base_angle = max(30, min(150, arm_base_angle))
        arm_elbow_angle = max(30, min(150, arm_elbow_angle))
        
        # 发送真实硬件指令
        set_servo_angle(1, int(arm_base_angle))
        set_servo_angle(2, int(arm_elbow_angle))
        set_servo_angle(3, 180 if gripper_state else 90) # 假设180是抓紧，90是松开
        
        time.sleep(0.05) # 20Hz控制频率

if __name__ == "__main__":
    main()
```

### 3.3 见证实物运动
在 RDK X5 的终端中运行你刚写的脚本：
```bash
python custom_arm_control.py
```
**实战效果**：当你推动 Joy-Con 左摇杆，现实中的大臂就会抬起或落下；推右摇杆，小臂跟着动；按下扳机键，夹爪就会夹紧！
配合 RDK X5 挂载的 USB 摄像头，你现在完全可以把手柄数据和摄像头画面同步保存下来，这便是一套纯手工打造的**低成本具身智能数据采集台**！
