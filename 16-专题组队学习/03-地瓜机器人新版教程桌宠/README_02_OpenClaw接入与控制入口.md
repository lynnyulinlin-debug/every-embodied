# OpenClaw 接入与控制入口

这一章说明桌宠教程中 OpenClaw、飞书会话和 MagicBox 控制命令之间的关系。读者完成本章后，应该能判断一次控制请求应该走飞书入口、OpenClaw Gateway，还是直接 SSH 到开发板。

## 控制链路

桌宠控制可以分成三层：

1. 飞书会话：用于发送自然语言指令和获取回复
2. OpenClaw Gateway：负责把会话请求转成工具调用或后端动作
3. MagicBox 板端命令：实际控制灯效、按钮、语音和舵机

## 什么时候用飞书

适合飞书入口的任务：

- 让桌宠播报一句话
- 触发一个预设灯效
- 查询当前会话状态
- 让 OpenClaw agent 解释下一步动作

如果飞书提示缺少 `sessionKey`，需要让读者复用当前教程中约定的会话标识，不要随机换一个新值。

## 什么时候用 SSH

适合直接 SSH 的任务：

- 检查板端服务是否真的运行
- 调试按钮、风扇、语音链路
- 验证 `magicboxctl` 是否存在
- 飞书入口不可用时做兜底控制

常用入口：

```bash
ssh sunrise@192.168.127.10
sudo /userdata/magicclaw/runtime/bin/magicboxctl status
```

## 发布时保留什么

保留读者能直接复用的命令和说明即可。不要提交本机临时脚本、完整运行日志、板端缓存和下载包。
