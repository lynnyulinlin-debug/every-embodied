# 05_VLA模型演化树与核心差异脉络：从开创到极致分化

本章综述中提到了多达 12 款前沿的 VLA / VLM 模型（从 RT-1 到 GR00T，再到 HoloBrain-0）。初学者很容易被这些繁杂的命名绕晕。
实际上，**这些模型之间绝非孤立存在，它们是一棵不断分化、相互填坑的“进化树”**。每一次新架构的提出，都是为了解决上一代模型留下的某个“致命缺陷（绝症）”。

以下我们从 **5 个核心技术维度**，为你梳理这些模型之间的**血脉联系**与**本质区别**。

---

## 维度一：动作输出的路线之争（离散 Token vs 连续 Diffusion）

这是 VLA 领域最大的帮派之争：机械臂的动作到底该怎么输出？

*   **开派宗师（离散化一派）：RT-1, RT-2, RT-X, OpenVLA**
    *   **联系：** 它们一脉相承，都坚持**“动作词元化（Action Tokenization）”**。把机械臂坐标强行切成 256 份，当作外语单词，让大模型玩 Next-Token Prediction。
    *   **区别（痛点）：** 这种做法实现简单，能直接复用 LLM 的基础设施。但**精度极差**（存在量化误差），做不了穿针引线等精细活，动作容易卡顿。
*   **革命者（连续扩散一派）：Octo, SwiftVLA, GR00T N1.5/N1.6**
    *   **联系与区别：** 它们抛弃了离散 Token，网络末端接的是 **扩散模型（Diffusion）** 或 **流匹配（Flow Matching）**。它们输出的是连续的浮点数轨迹。
    *   **结果：** 动作极其丝滑平顺，精度达到毫米级。Octo 是纯动作模仿的王者，GR00T 则将其做到了工业级的平滑。

## 维度二：“脑力”与推理深度的演进（从条件反射到 System 2思考）

大模型到底在控制中扮演什么角色？是单纯的翻译机，还是懂物理的哲学家？

*   **无脑肌肉男：Octo**
    *   **特点：** 彻底阉割了语言大模型（没有 LLM），纯靠图像输入直接输出动作。它的区别在于**完全没有对话和逻辑推理能力**，但换来的是极致的推理速度。
*   **基础指令跟随：RT-1, OpenVLA, SmolVLA**
    *   **特点：** 具备基础的语言理解（如“把苹果放进碗里”）。它们属于 **“条件反射（System 1）”**，看图->直接输出动作。
*   **常识与深层推理（System 2）：RT-2, GR00T N1.6**
    *   **特点：** 它们接入了巨型 VLM（如 55B 的 PaLM-E 或 Cosmos Reason）。
    *   **本质区别：** 当你给出一个模糊指令“我把水洒了”，前面的模型会宕机，而 RT-2 和 GR00T N1.6 会先进行**思维链（CoT）推演**：“水洒了 -> 需要吸水的东西 -> 寻找海绵 -> 抓取海绵”。GR00T N1.6 更是加入了物理常识的沙盘推演。

## 维度三：感知维度的升维（从 2D 扁平到 3D 物理显式）

早期模型看世界就像看一张平面的照片，这导致它们在不同视角的相机间迁移时（Sim-to-Real）极易翻车。

*   **2D 像素流派：RT 系列, OpenVLA, Octo**
    *   **特点：** 直接把 2D RGB 图片喂给 ViT（视觉 Transformer）。它们根本不知道什么叫“深度”，纯靠死记硬背像素和动作的映射。
*   **3D 空间觉醒：SpatialVLA**
    *   **区别：** 强行注入了 Ego3D 深度位置编码。它看世界不再是扁平的，而是立体的网格，大大提高了插入、精细抓取的成功率。
*   **显式物理流派（终极解法）：HoloBrain-0**
    *   **本质区别：** 它是唯一一个把机器人自身的“筋骨（URDF 运动学文件）”和“相机位置（外参矩阵）”作为输入喂给大模型的。它解决了跨越不同机械臂形态（Cross-Embodiment）时的水土不服问题。

## 维度四：时间轴的掌控（从“金鱼记忆”到“预判未来”）

抓取不是一瞬间的事，而是一个连续的视频流。

*   **单帧失忆症：OpenVLA, RT-1**
    *   **特点：** 每次只看当前这一帧图片做决策。在执行“打开抽屉拿苹果再关上抽屉”的长线任务时，中间很容易忘记自己原来要干嘛。
*   **向过去借记忆：RoboFlamingo**
    *   **区别：** 引入了历史多帧图文记忆。它在网络里保留了过去的画面特征，彻底治好了长线任务中的失忆症。
*   **向未来做预判：SwiftVLA**
    *   **区别：** 如果目标物体（如传送带上的快递）在移动，前面的模型都会抓空。SwiftVLA 引入了 4D 时空动态预测，它能在脑海中“预判”机械臂和物体在未来几秒的相遇轨迹。

## 维度五：落地门槛与工程分化（云端巨兽 vs 边缘部署）

学术界追求上限，工业界追求落地。

*   **云端算力黑洞：RT-2 (55B), GR00T, StarVLA (7B)**
    *   **特点：** 需要极其庞大的算力集群支撑。StarVLA 更是提出将海量互联网图文数据与机器人动作数据放在同一个熔炉里协同训练（Co-training），这是只有大厂才能玩得起的炼丹炉。
*   **平民与边缘王者：SmolVLA (450M), SwiftVLA (450M), HoloBrain-0 (183M)**
    *   **联系与区别：** 它们是算力贫民窟的救星。通过特征压缩、网络层剪枝，将参数量压缩到了不足 0.5B。这使得它们可以直接在国产边缘计算板（如 RDK-X5 或树莓派）上以 10Hz+ 的频率流畅运行，而不需要依赖远端的 RTX 4090 服务器。

---

## 总结：如何根据需求选择你的 VLA？

这张演化树也为我们提供了明确的**技术选型指南**：

1.  **你想快速复现一个 baseline，且有 24G 显卡：** 首选 **OpenVLA (7B)**。生态最好，开箱即用。
2.  **你不需要机器人听懂人话，只求动作又快又准（如工业流水线）：** 首选 **Octo**。纯动作扩散，极速推理。
3.  **你的机器人要处理复杂的长流程家务：** 选择 **RoboFlamingo**（带历史记忆）或接入带 CoT 推理的 VLM。
4.  **你的机器人要在流水线上抓取移动物体：** 必须使用具备时空预测能力的 **SwiftVLA**。
5.  **你想在百元级的开发板上离线跑大模型：** 毫不犹豫地选择 **SmolVLA** 或 **HoloBrain-0** 轻量版。
